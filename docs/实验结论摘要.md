# 实验结论摘要

Demo1 说明：低比特量化只减少权重存储，不会自动带来推理加速。PyTorch naive INT4 会显式生成 `W_dequant`，中间张量和显存带宽开销很大；Triton fused INT4 通过 kernel 内融合避免完整反量化矩阵写回，因此相对 naive INT4 有明显加速。

Demo2 说明：`torch.compile` 能提供一定自动优化能力，适合作为 baseline；但 packed INT4 的关键在于数据格式控制和 kernel 内融合，手写 Triton 能直接表达这些操作，因此在已有测试规模下比 `torch.compile` 更快。

Demo3 说明：Triton MXFP-like fused 相对 Full naive MXFP4 有约 44.5x 到 91.9x 的加速，但 RTX 3060 不支持原生 MXFP4 block-scaled Tensor Core，所以结果只代表软件融合下限，不代表 Blackwell 等硬件上的原生 MXFP4 上限。
